para ejecutar estos nuevos cambios deberá hacerse algunas configuraciones en su maquina,  saber que no podrás ejecutar la aplicación en iPhone si no se dispone de un dispositivo macOX con xcode y cocoapod previamente configurado.
****Este login aun no esta soportado para ios, ya que se requiere hacer un pago previo para utilizar el componente*****

 - debes crear una SHA1 para tu equipo, no para el proyecto. no sobrescribas el que ya tiene el proyecto porque eso haría que deje de funcionar el login.

- para crear esta firma se puede hacer con la herramienta de java keytool, mira la documentación para instalarla si aun no lo tienes instalado, recomiendo la versión de java 17.  debes ejecutar el siguiente comando: keytool -genkey -v -keystore C:\Users\”NOMBRE DE USUARIO DE TU PC”\.android\debug.keystore -alias AndroidDebugKey -keyalg RSA -keysize 2048 -validity 10000.  esto te pedirá que introduzcas la contraseña del ordenador en caso de que no la ejecutes como administrador y te pedira otra contraseña para la clave, inserta la contraseña: android 

- haz un cd android en la consola del proyecto y ejecuta el comando: ./gradlew signingReport. este comando leerá las key del proyecto y de tu ordenador, el que te interesa la que esta en tu ordenador lo podras identificar por la ruta de la propiedad Store la cual te debe indicar que se obtiene del usuario del ordenador 

- luego que ya tengas la keystore deberás añadirla a la app de android en firebase, en la consola de firebase selecciona el proyecto y en la configuración del proyecto en la parte de abajo podrás ver la app de android, aquí debes añadir la clave SHA1 que acabas de crear
